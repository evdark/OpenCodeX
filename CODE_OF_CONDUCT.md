# Code of Conduct

OpenCodeX should be a useful, serious, and welcoming place to build. We expect contributors to treat each other with professionalism and patience, especially when discussing bugs, design tradeoffs, and upstream compatibility.

## Expected Behavior

- Be respectful and direct.
- Assume good intent while asking for clarity.
- Keep feedback specific to the work.
- Credit prior work and upstream contributions.
- Make room for new contributors to learn the codebase.

## Unacceptable Behavior

- Harassment, discrimination, threats, or personal attacks.
- Dismissing people instead of addressing technical points.
- Posting private information without permission.
- Repeatedly derailing issues, pull requests, or discussions.
- Abusing issue templates, security reports, or maintainer time.

## Enforcement

Maintainers may edit, hide, or remove comments; close issues or pull requests; block accounts; or limit participation when behavior harms the project.

If you need to report a conduct issue, contact a maintainer privately through GitHub. Security issues should follow [SECURITY.md](SECURITY.md).

## Scope

This code of conduct applies to OpenCodeX repository spaces, issues, pull requests, discussions, and any official community channels linked from this repository.
