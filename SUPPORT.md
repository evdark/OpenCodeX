# Support

OpenCodeX support is organized by urgency and privacy.

## Questions

Use GitHub Discussions for:

- setup questions;
- provider configuration help;
- plugin and MCP guidance;
- roadmap feedback;
- workflow ideas.

If Discussions are not enabled yet, open a `Question` issue using the template.

## Bugs

Open a bug report when something is broken or surprising. Include:

- version or commit;
- operating system;
- install method;
- provider and model, if relevant;
- reproduction steps;
- logs or screenshots with secrets removed.

## Feature Requests

Open a feature request for focused product improvements. Describe the workflow, why it matters, and what a good first version would do.

## Security

Do not open public issues for vulnerabilities. Follow [SECURITY.md](SECURITY.md).

## Upstream OpenCode

OpenCodeX is an unofficial fork. If your issue reproduces only in official OpenCode, report it upstream. If it reproduces in both projects, mention that in the issue so maintainers can decide where the fix belongs.
