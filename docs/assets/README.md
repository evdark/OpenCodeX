# Visual Assets

This directory keeps OpenCode+ repository-facing visuals in one place so the root stays clean and GitHub pages remain easy to scan.

## Structure

| Path           | Purpose                                                       |
| -------------- | ------------------------------------------------------------- |
| `brand/`       | Fork-specific marks, previews, and future social card assets. |
| `screenshots/` | Curated screenshots used by README and documentation pages.   |

## Rules

- Keep fork-specific assets under `docs/assets` instead of the repository root.
- Reuse upstream package assets by linking to them; avoid moving upstream-owned files.
- Prefer descriptive file names such as `desktop-settings-uk.png`.
- Replace temporary screenshots with current OpenCode+ captures before public releases.
